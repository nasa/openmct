limitEvaluator(domainObject) → {module:openmct.TelemetryAPI~LimitEvaluator}

Get a limit evaluator for this domain object. Limit Evaluators help you evaluate limit and alarm status of individual telemetry datums for display purposes without having to interact directly with the Limit API.

This method is optional. If a provider does not implement this method, it is presumed that no limits are defined for this domain object's telemetry.

TelemetryProperty

Describes a property which would be found in a datum of telemetry associated with a particular domain object.

Properties:
Name Type Attributes Description
key string

the name of the property in the datum which contains this telemetry value
name string

the human-readable name for this property
units string <optional>

the units associated with this property
temporal boolean <optional>

true if this property is a timestamp, or may be otherwise used to order telemetry in a time-like fashion; default is false
numeric boolean <optional>

true if the values for this property can be interpreted plainly as numbers; default is true
enumerated boolean <optional>

true if this property may have only certain specific values; default is false
values string <optional>

for enumerated states, an ordered list of possible values

TelemetryRequest

Describes and bounds requests for telemetry data.

Properties:
Name Type Description

the key of the property to sort by. This may be prefixed with a "+" or a "-" sign to sort in ascending or descending order respectively. If no prefix is present, ascending order will be used.
start *

end *

the upper bound for values of the sorting property
