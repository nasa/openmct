Interface: CompositionProvider

CompositionProvider

A CompositionProvider provides the underlying implementation of composition-related behavior for certain types of domain object.

By default, a composition provider will not support composition modification. You can add support for mutation of composition by defining add and/or remove methods.

If the composition of an object can change over time-- perhaps via server updates or mutation via the add/remove methods, then one must trigger events as necessary.
